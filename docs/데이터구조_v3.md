# 키워팜 v3 데이터 구조 정리 (2026-06-04 v2 — 텃밭 큐레이션 반영)

> 변경 요약: 농사로 키 단일화(NONGSARO_API_KEY 하나), weekfarm 폐기,
> monthtech 텃밭 관련성 선별(-70%), 텃밭가꾸기(garden)·이달의음식(monthfd) 신규 임베딩.

## 1. 핵심 원칙: 작물 ID는 슬러그 하나로

모든 작물 데이터는 **`backend/data/crops_master.json`의 40종 영문 슬러그**(`lettuce`, `basil`, `cherry_tomato`...)를 유일한 키로 쓴다. KAMIS 코드(214, 223...)는 레거시이며, 호환 레이어(`ingest.crop_key`)가 자동 변환하므로 **새 코드에서는 슬러그만 쓰면 된다.**

## 2. 데이터 폴더 현황 (`backend/data/`)

`_common.garden` — 작물 불문 텃밭 공통 가이드(물주기·밭 만들기·병해충 관리·용기 준비 등 ~40건).
챗봇에서 작물 키 검색과 함께 `retrieve("_common", 질의)` 를 병행하면 일반 질문을 커버한다.

## 3. 팀원별 사용법

### AI 추천 + 챗봇 담당
- 작물 메타·캘린더: `core/planting/matrix.py` — `all_crops()`, `get_crop(slug)`, `actions_in_month()`
- **챗봇 RAG**: `app/core/rag/retrieve.py` — `retrieve("lettuce", 질의)` 또는 kind 가중치를 주는 `retrieve_boosted()`. 슬러그로 호출하면 그 작물의 모든 지식(cultivation+monthtech+weekfarm+general)이 검색됨
- 한글 작물명 → 마스터 레코드: `app/data/crop_ids.py`의 `find_by_name("바질")` — 별칭 처리 포함(총각무→알타리무, 풋고추→고추 등, `NAME_ALIAS`)
- ⚠️ **40종 중 ~17종(쑥갓·루꼴라·바질·페퍼민트·로즈마리·콜라비 등)은 임베딩이 아직 없음** → `retrieve`가 빈 리스트 반환. 농사로 텃밭가꾸기(fildMnfct)·실내정원(garden) 수집으로 채우는 작업 필요

### 기르기 일지 + 캘린더 담당
- **기존 코드 수정 불필요.** `knowledge.get_calendar_tasks(item_code, kind_code, ...)`를 KAMIS 코드로 호출해도 `ingest.crop_key()`가 슬러그로 자동 정규화해 새 파일을 읽는다 (검증 완료: `crop_key("214","00") → "lettuce"`)
- **신규 작업부터는 슬러그 직접 사용 권장** — `crop_key("lettuce", "")`도 그대로 동작. `farm_plan` DB의 `crop_item_code`에 슬러그를 넣는 방향으로 점진 전환하면 호환 레이어를 나중에 걷어낼 수 있음
- 월별 작업은 `matrix.json`(`actions_in_month()`)이 1차 소스, RAG는 보강용

### 수확 인증 카드 담당
- `scripts/sync_month_fd.py` — 이달의음식 수집 → 작물별 분류(`data/monthfd/by_crop.json`) → `{슬러그}.monthfd` 임베딩 (수집 완료: 18작물 392청크. 레시피는 음식명 부분일치로 다작물 연결)
- 수확카드는 `monthfd` kind만 명시 로드 — 추천·캘린더 검색 결과에 섞이지 않음

## 4. 팀 규칙

1. **새 작물 데이터는 무조건 슬러그 키로 저장** — KAMIS 코드 신규 사용 금지
2. **40종 외 작물 데이터는 추가하지 않음** — 작물 추가가 필요하면 `crops_master.json`에 먼저 등재
3. 임베딩 쓰기: `store.save(슬러그, kind, chunks, vectors, source)` — 용도별로 kind를 분리하면 `retrieve_boosted`로 출처별 가중치 검색 가능
4. 레거시 매핑·별칭 변경은 `app/data/crop_ids.py` 한 곳에서만
